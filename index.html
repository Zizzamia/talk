#ngTasty

Building high performance measurable directives

by Leonardo Zizzamia

@zizzamia

In twitter we have a collection of UI component called start as fork of my personal project ngTasty

The main goal was implement something

  • easy to reuse
  • easy to test
  • easy to benchmark

jsPerf & Benchpress

But there is a lot of manual work to do, to create test version by version

Now I want show you the way we do in ngTasty

We have a Yeoman generator help us to create benchmark

You can also create benchmark of specific bower version

Show benchmark

Here I’m creating a benchmark of three main table directive, ng-grid / ng-table / ng-tasty

Building the benchmark

As you can see now we can easily understand performance between table.

ng-grid is 50ms, ng-table 100ms, ng-tasty 18ms

Let’s something way more cool, like how everyone can improve the performance by disabling the log

Running the benchmark

What about if I’m debugging my directive and see the difference in run-time, we notice that to run a benchmark you need call

  • benchpress build
  • benchpress run

We can use this `gulp benchpress` and will watch all the change of dist, and update the static assets of the benchmark
